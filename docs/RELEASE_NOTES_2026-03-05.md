# Release Notes - 2026-03-05

## Что зафиксировано

- Проект переведен на модель без курьеров: клиент делает заказ, вендор доставляет сам.
- Разделены Telegram-боты:
  - клиентский бот
  - вендорский бот
- Поддержана авторизация и синхронизация клиента через Telegram Mini App.
- Добавлен перенос заказов/адресов/промо данных со старых dev-id на реальный `tg:<telegram_user_id>`.
- Обновлена работа клиентского Mini App:
  - стабилизация старта приложения в Telegram WebView
  - fallback идентификации через launch-параметры
  - исправления для профиля и заказов
- Обновлена вендорка:
  - мобильная адаптация ключевых экранов
  - скрыт лишний vendor id в шапке
  - восстановлен интерактивный выбор координат
- Обновлена админка:
  - добавлен выход из аккаунта
  - улучшены рабочие экраны клиентов/вендоров

## Текущие dev-команды

- API: `npm run dev:api`
- Client app: `npm run dev:client`
- Vendor app: `npm run dev:vendor`
- Client bot: `npm run dev:bot:client`
- Vendor bot: `npm run dev:bot:vendor`

## Важные замечания

- Для Telegram Mini App требуется корректный `initData`.
- Если используется Cloudflare Quick Tunnel, ссылки временные и после сна/перезапуска могут меняться.
- Для стабильного теста нужны одновременно запущенные API, клиент/вендор фронты и соответствующие боты.

## Что еще проверить после деплоя

- Создание клиента с нового Telegram-аккаунта и изоляция заказов между аккаунтами.
- Стабильность карты и сохранения адреса в клиентке/вендорке.
- Полный флоу заказа: создание -> принятие -> доставка -> завершение.
- Проверка отмены заказа (вручную и по таймауту).
