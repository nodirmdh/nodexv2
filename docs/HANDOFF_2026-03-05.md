# Handoff - 2026-03-05

## Purpose
This file is a restart point for continuing development after a long pause.

## Product direction locked for MVP
- Courier flow is out of active runtime.
- Delivery model: client places order, vendor delivers.
- Admin stays web.
- Client and Vendor are Telegram Mini Apps.
- Separate Telegram bots for client and vendor entry points.

## Implemented in this iteration

  - API resolves client identity from Telegram source first.
  - Sync endpoint can merge legacy dev clients into real `tg:<telegram_user_id>`.
- Client mini app auth/header flow hardened:
  - Stable `x-role: CLIENT`.
  - Safer token handling and fallback logic.
- Client startup reliability improved:
  - Reduced fatal startup crash path in Telegram WebView.
  - Better handling for stale active order response states.
- Vendor mini app:
  - Interactive map selection restored.
  - Removed unnecessary visible vendor ID field in top area.
- Admin:
  - Logout action exposed in UI.
  - Vendor details map behavior improved.
- Telegram bot:
  - `/start` client flow requests phone first, then shows Open Mini App button.
  - Launch params include Telegram user fields for fallback usage.

## Known unstable areas
- Intermittent `Script error` in client mini app can still appear in Telegram WebView after some interactive actions.
- `initData` missing/expired situations still need stricter UX fallback messaging and retry guidance.
- Quick tunnel links are temporary and expire after sleep/restart.

## Runtime checklist (local)
1. `npm install`
2. Ensure `.env` is filled in root and app env files.
3. Start API: `npm run dev:api`
4. Start client web: `npm run dev:client`
5. Start vendor web: `npm run dev:vendor`
6. Start bots:
   - `npm run dev:bot:client`
   - `npm run dev:bot:vendor`
7. If needed, recreate Cloudflare quick tunnels and update:
   - `TELEGRAM_CLIENT_WEBAPP_URL`
   - `TELEGRAM_VENDOR_WEBAPP_URL`
   - `VITE_API_URL` in client/vendor apps

## Operations notes
- If Telegram bot throws `409 Conflict` on polling, another instance is running: stop duplicates and run one process per bot token.
- If API says `EADDRINUSE:3000`, stop old process on port 3000.
- If mini app shows blocked host on Vite, add tunnel host into `server.allowedHosts` (or use preview/static mode).

## Data migration caution
- Do not delete legacy clients blindly; some old orders may still reference them.
- Migration should move relational data first (`orders`, `addresses`, promo relations), then remove empty legacy rows.
